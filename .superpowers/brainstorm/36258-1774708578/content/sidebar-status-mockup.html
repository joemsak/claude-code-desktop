Status States

Idle
Waiting for input (Stop hook fired)
Working
Generating / using tools (PreToolUse / PostToolUse)
Waiting
Needs your attention (Notification hook)
Exited
Process ended (PTY exit event)

Key Difference from Previous Attempt

The old attention indicator used a 2-second timeout heuristic — "if output stops, assume Claude is waiting." This fired on idle tabs, startup output, and any pause in output.

The new approach uses Claude Code's built-in hooks API — real lifecycle events that fire at exact state transitions. The Stop hook fires precisely when Claude finishes a response. The Notification hook fires precisely when Claude needs user input. No guessing.
